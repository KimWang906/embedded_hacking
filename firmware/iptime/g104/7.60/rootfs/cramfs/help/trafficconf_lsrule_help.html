• 통신사(ISP)별 분산규칙
    • 목적지 IP주소를 관리하는 통신사(ISP)를 판단하여 지정한 WAN 포트로 트래픽을 분산 시킬 수 있는 기능입니다.
    • (예) KT와 SK브로드밴드에 인터넷 서비스를 각각 WAN1과 WAN2에 연결하여 사용하는 경우

      • - KT를 WAN1으로 지정하고, SK브로드밴드를 WAN2로 지정하여 설정합니다.
      - 해당 설정에 따라 공유기는 패킷의 목적지 IP주소를 보고 KT 또는 SK브로드밴드에서 관리되는 IP주소 인지를 확인하여 지정된 WAN으로 분산시킵니다.
      - 패킷의 목적지 IP주소가 설정한 어떤 통신사(ISP)에도 매치가 안되는 경우에는 '분산정책 기본설정'과 '사용자정의 분산규칙'에 따라 분산 됩니다.
    • '사용자정의' 파일 형식

      • - 일반적인 텍스트 문서 파일로서 별도의 확장자 구분은 없습니다.
      - 100.100.100.1~100.100.100.200 또는 100.100.100.234 와 같이 IP주소범위 또는 단일IP주소를 한줄씩 입력하시면 됩니다.
      - 반드시 한줄에는 하나의 IP주소범위 또는 단일IP주소만을 입력하셔야 합니다.
      - 그렇지 않는 경우에는 각 줄의 맨앞의 IP주소만을 처리합니다.

  • 사용자정의 분산규칙
    • 분산정책은 사용자가 정의한 인터넷 트래픽을 지정한 WAN 포트로 분산시킬 수 있는 기능입니다.
    • 먼저 입력한 규칙의 우선 순위가 가장 낮습니다.
  • 규칙이름
    • 임의로 입력가능하며 구분이 쉽도록 입력합니다.
  • 내부 PC IP 주소
    • 분산시키고자 하는 PC의 IP주소를 입력합니다.
  • 목적지 IP 주소
    • 분산시키고자 하는 목적지의 IP주소를 입력합니다.
  • 프로토콜
    • 프로토콜 타입을 (any/TCP/UDP)을 선택합니다.
    • 'any'는 모든 프로토콜을 의미합니다. 'any' 선택시에는 포트번호를 입력할 수 없습니다.
  • 시작지 포트
    • 분산시키고자 하는 packet의 시작지(Source) 포트 번호를 입력합니다.
  • 목적지 포트
    • 분산시키고자 하는 packet의 목적지(Destination) 포트 번호를 입력합니다.
  • 분산경로
    • 분산 경로에는 정의한 트랙픽이 라우팅될 WAN 포트를 선택합니다.

  • 규칙간 우선순위
    • 자동 분산 < 통신사(ISP)별 분산 < 사용자정의 분산
