• 개 요
    • 이 기능을 이용하면 한개의 AP로 여러개의 무선 네트워크를 구성할 수 있습니다.
    • 기본 무선 네트워크는 [ 무선 설정/보안 ] 에서 설정된 무선 네트워크를 의미합니다.
    • 이 기능은 기본 무선 네트워크가 AP모드로 실행 중일 경우만 동작합니다.
    • 각각의 무선 네트워크에 독립적인 인증/암호화 설정이 가능합니다.
    • 모드,채널,지역 설정은 기본 무선 네트워크와 동일합니다.
  • 사용 방법
    • 추가될 무선 네트워크 정보를 입력 후, 추가 버튼을 누릅니다.
    • 추가된 리스트를 클릭하면, 해당 줄이 반전되면서 수정 모드로 바뀝니다.
    • 취소 버튼을 누르면, 수정모드를 취소합니다.
    • 무선 랜카드를 가진 PC에서 추가된 무선 네트워크가 주변에 존재하는지 확인합니다.
  • 설정 인자
    • 무선 네트워크 이름
      • 추가될 무선 네트워크 이름을 입력합니다.
    • 통신 정책
      • 모든 통신 허용 - 본 공유기를 통한 인터넷 및 내부 네트워크로의 통신을 허용합니다.
      • 인터넷 통신만 허용 - 인터넷 통신만 허용합니다. 내부 네트워크로의 통신은 허용하지 않습니다.
      • 내부 네트워크 통신만 허용 - 내부 네트워크로의 통신만 허용합니다.
      • 내부 네트워크란 유선 랜포트로 연결된 네트워크,기본 무선 네트워크,타 무선 네트워크를 의미합니다.
    • SSID(네트워크이름)알림
      • 추가될 무선 네트워크를 숨길 수 잇습니다.
      • 이 기능을 사용하지 않을 경우 네트워크 이름을 알아야만 접속할 수 있습니다.

    • 암호화 방법
      • 보안 암호화 설정을 통하여 허가된 사용자만이 무선랜을 통하여 접속할 수 있습니다.
      • 인증방법 : 무선 사용자가 암호화된 인증을 통하여 접속이 가능하게 할 수 있습니다.
        • 개방모드 - 암호화된 인증을 사용하지 않습니다.
        • 공유모드 - 설정된 Key 값을 사용하여 암호화된 인증을 합니다.
            (WEP64와 WEP128의 암호화 방법을 사용할 수 있습니다.)
        • 자동 - 위의 두가지 경우를 모두 지원합니다.
        • WPAPSK, WPA2PSK - 802.11i에 근거한 보다 향상된 인증방법을 사용합니다.
            (TKIP과 AES의 암호화 방법을 사용할 수 있습니다.)
        • ※ WPAPSK/WPA2PSK 사용시에는 두가지 인증방법으로 모두 접속할 수 있습니다.
      • 암호화 방법 : 무선 통신의 보안을 위하여 데이타를 암호화 합니다.
        • 사용안함: 암호화를 사용하지 않습니다.
        • WEP 64 : 64Bit Key를 사용하여 암호화 합니다.
        • WEP 128 : 128Bit Key를 사용하여 암호화 합니다.
        • TKIP, AES : 8자 이상의 문자열을 입력 합니다.
      • 입력 방법 : 암호화에 사용되는 KEY를 입력하는 방법을 선택합니다.
        • 문자열 : 문자열로 입력합니다.
        • 16진수 : 16진수로 입력합니다.
        • KEY 입력 규칙 :
          • 하나의 KEY는 모든 칸을 채워서 입력해야 합니다.
          • 기본 KEY는 항상 입력해야합니다.
          • 64Bit-문자열 :5개 , 128Bit-문자열 : 13개
          • 64Bit-16진수 :10개 , 128Bit-16진수 : 26개 ( 숫자 및 A-F 영문자만 입력가능 )
