• 기본 분산 경로
    • 별도의 규칙으로 지정되어 있지 않은 인터넷 트래픽은 기본 분산 경로로 통신이 이루어진다.

  • 자동 인터넷 라인 백업
    • WAN1 또는 WAN2 중 임의의 WAN포트에서 장애가 발생하면 정상 동작하는 WAN 포트로 모든 통신이 자동으로 이루어 집니다.
    • 라인 백업 확인 주기 : 설정 주기동안 각 WAN별 게이트웨이와의 통신을 확인 합니다"

  • 자동 분산 비율 설정
    • 설정된 비율대로 인터넷 트래픽을 자동 분산 시킵니다.
    • PC IP 주소 단위 분산

      • PC IP 주소를 기준으로 분산.
      [STATUS]를 클릭하면 현재 PC IP주소별 통신하는 WAN 주소를 확인 할 수 있습니다.
      단, 사용자정의 분산정책에서 'PC IP주소' 옵션값으로만 설정된 PC IP주소는 표시되지 않습니다.
      예) {192.168.0.100} - WAN1, {192.168.0.200} - WAN2
    • PC IP 주소 + 포트단위 분산

      • {PC IP 주소, 소스 포트 번호} 조합 기준으로 분산.
      즉, 한대의 PC에서 새로운 통신이 이루어 질때 마다 사용되는 WAN이 변경 될 수 있음.
      예) {192.168.0.100, 1234} - WAN1, {192.168.0.100, 4321} - WAN2

  • 사용자정의 분산정책
    • 분산정책은 사용자가 정의한 인터넷 트래픽을 지정한 WAN 포트로 분산시킬 수 있는 기능입니다.
    • 먼저 입력한 규칙의 우선 순위가 가장 낮습니다.
  • 규칙이름
    • 임의로 입력가능하며 구분이 쉽도록 입력합니다.
  • 내부 PC IP 주소
    • 분산시키고자 하는 PC의 IP주소를 입력합니다.
  • 목적지 IP 주소
    • 분산시키고자 하는 목적지의 IP주소를 입력합니다.
  • 프로토콜
    • 프로토콜 타입을 (any/TCP/UDP)을 선택합니다.
    • 'any'는 모든 프로토콜을 의미합니다. 'any' 선택시에는 포트번호를 입력할 수 없습니다.
  • 시작지 포트
    • 분산시키고자 하는 packet의 시작지(Source) 포트 번호를 입력합니다.
  • 목적지 포트
    • 분산시키고자 하는 packet의 목적지(Destination) 포트 번호를 입력합니다.
  • 분산경로
    • 분산 경로에는 정의한 트랙픽이 라우팅될 WAN 포트를 선택합니다.
